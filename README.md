
# YouTrack Performance Testing with JMeter

Goal of this repository is to provide results of tasks - running performance tests on YouTrack system using JMeter, analyzing the results, and generating visual plots and tables for better understanding of the system's performance under different load scenarios.

Goals, requirements, methodology and approach for performance testing is described in the report. [REPORT](https://docs.google.com/document/d/16-L1Zk5R_DaznVoAo4jKk57-VrrkSo2KibZP6ogbjK4/edit?usp=sharing) located in google docs contains the detailed analysis of the performance testing results, including the interpretation of the generated plots and tables, and conclusions about the system's performance under different load scenarios.

### Used tools and technologies:`
- [JMeter v5.6.3](https://jmeter.apache.org/) - an open-source tool for performance testing and load testing of web applications, used to simulate different load scenarios on the YouTrack system and generate performance test results in .jtl format. used scenarios are located in `jmeter_scenarios` folder
- [R](https://www.r-project.org/) - a programming language and environment for statistical. R scripts are used to process, analyze, and visualize the JMeter performance test results, generating plots and tables for better understanding of the system's performance under different load scenarios. R scripts are located in `Src` folder
- [YouTrack v 2026.1.13570](https://hub.docker.com/r/jetbrains/youtrack/tags/?name=2026.1.13570) - a web-based issue tracking and project management tool, the system under test for performance testing. Scripts for YouTrack container deployment and system initialization are located in `Script` folder

# Folders description:
- `jmeter_results` - contains .jtl files generated by JMeter after running the tests, also contains the cleaned and combined data in .rds format and tables generated by the analysis script
  - `all_scenarios_dt.rds` - the cleaned and combined data from all JMeter .jtl files, saved in RDS format for analysis and visualization
  - `jmeter_resultls.zip` - contains the raw .jtl files generated by JMeter for each test scenario. it was zipped to reduce the size of the repository, you can unzip it to access the individual .jtl files for each scenario
- `jmeter_reports` - contains the generated HTML reports from JMeter and the visual plots generated by R scripts
  - `ggplot` - contains the visual plots generated by R scripts for performance analysis
  - `tables` - contains the generated tables from the analysis script
  - `jmeter_html_reports` - contains the HTML reports generated by JMeter for each test scenario
- `jmeter_scenarios` - contains the JMeter test plans (.jmx files) for different test scenarios
  - `01_create_users.jmx` - JMeter test plan for creating users in YouTrack
  - `02_create_issues.jmx` - JMeter test plan for creating issues in YouTrack
  - `03_scenario_default.jmx` - JMeter test plan for the default scenario expected to generate 40 actions per user during the hour by 100 active concurrent users with in gaussian distribution of actions per user between user activities.
  - `04_scenario_step.jmx` - JMeter test plan for the step increasing load by 10 action user each 5 minutes without gaussian distribution of actions per user between user activities, simulate higher load on the system and observe how it handles the increasing number of active users and actions over 1 hour scenario.
- `Script` - [Script](Script/readme.md) contains the shell scripts for YouTrack container deployment and system initialization
- `Src` -  [Src](Src/readme.md) contains the R scripts for processing, analyzing, and visualizing JMeter performance test results

# Workflow for performance testing and analysis:
1. Deploy YouTrack system using the provided shell scripts in the `Script` folder
2. Run JMeter tests using the provided test plans in the `jmeter_scenarios` folder
3. Analyze the generated .jtl files using the provided R scripts in the `Src` folder to generate visual plots and tables for better understanding of the system's performance under different load scenarios
4. Generate JMeter HTML reports for each test results using the provided commands in the `README.md` file


#### Generating JMeter HTML Reports
